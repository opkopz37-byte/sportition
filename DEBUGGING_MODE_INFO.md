# 🐛 Cursor AI 디버깅 모드란?

## 📌 디버깅 모드 개요

Cursor AI의 **디버깅 모드 (Debug Mode)**는 코드 실행 중 발생한 오류를 체계적으로 분석하고 해결하는 데 특화된 AI 모드입니다.

---

## 🎯 디버깅 모드의 주요 기능

### 1. 자동 오류 감지
- 런타임 에러 자동 인식
- 컴파일 에러 추적
- 예외 스택 트레이스 분석

### 2. 문제 원인 파악
- 에러 메시지 정밀 분석
- 관련 코드 섹션 자동 탐색
- 의존성 문제 확인

### 3. 해결책 제시
- 구체적인 수정 방안 제공
- 여러 해결 옵션 비교
- 단계별 수정 가이드

### 4. 코드 실행 추적
- 변수 값 변화 추적
- 함수 호출 흐름 분석
- 상태 변화 모니터링

---

## 🚀 디버깅 모드 사용 방법

### 방법 1: 에러 발생 시 자동 제안
1. 코드 실행 중 에러 발생
2. Cursor가 자동으로 디버깅 모드 제안
3. 제안 수락 시 디버깅 모드로 전환

### 방법 2: 수동 활성화
1. Cursor 좌측 하단의 **모드 선택** 버튼 클릭
2. **Debug Mode** 선택
3. 에러 로그나 문제 상황 설명

### 방법 3: 채팅에서 요청
```
"디버깅 모드로 전환해서 이 문제를 분석해줘"
```

---

## 💡 현재 상황에 디버깅 모드가 도움이 될까?

### ✅ 디버깅 모드가 유용한 경우

1. **런타임 에러**: 앱 실행 중 충돌이나 예외 발생
   - 예: "Cannot read property 'x' of undefined"
   - 예: "Network request failed"

2. **데이터 흐름 문제**: 데이터가 예상과 다르게 흐르는 경우
   - 예: 프로필 수정 후 대시보드에 반영 안 됨
   - 예: 출석 체크 후 스킬 포인트 증가 안 됨

3. **성능 문제**: 로딩이 느리거나 멈춤
   - 예: 페이지 로딩이 무한히 지속
   - 예: API 호출이 타임아웃

4. **SQL 에러**: Supabase 쿼리 실패
   - 예: "syntax error at or near..."
   - 예: "relation does not exist"

### ❌ 디버깅 모드가 불필요한 경우

1. **기능 추가 요청**: 새로운 기능 개발
   - 예: "회원가입에 이메일 인증 추가해줘"
   - → 일반 Agent 모드로 충분

2. **UI 개선 요청**: 디자인 변경
   - 예: "버튼 색상을 파란색으로 바꿔줘"
   - → 일반 Agent 모드로 충분

3. **코드 리뷰**: 잠재적 문제 탐색
   - 예: "코드 검수해줘"
   - → Ask 모드나 Agent 모드로 충분

## 🔍 현재 프로젝트에서 디버깅 모드 사용 예시

### 사례 1: SQL 에러 발생 시

**문제 상황:**
```
ERROR: 42601: syntax error at or near "#" LINE 15
```

**디버깅 모드 활용:**
1. 에러 메시지와 SQL 파일 제공
2. AI가 자동으로 문법 오류 위치 파악
3. 수정된 SQL 제공
4. 실행 가이드 제시

### 사례 2: 출석 체크가 작동하지 않을 때

**문제 상황:**
- 출석 체크 버튼 클릭해도 반응 없음
- 스킬 포인트 증가 안 됨

**디버깅 모드 활용:**
1. 브라우저 콘솔 로그 분석
2. `checkAttendance` 함수 실행 흐름 추적
3. Supabase 연결 상태 확인
4. 각 단계별 문제점 파악
5. 순차적 수정 방안 제시

### 사례 3: 프로필 정보가 대시보드에 반영 안 될 때

**문제 상황:**
- 프로필 수정 후 대시보드가 업데이트되지 않음

**디버깅 모드 활용:**
1. `refreshProfile()` 호출 여부 확인
2. `useEffect` 의존성 배열 검사
3. 상태 업데이트 흐름 추적
4. `AuthContext` → `dashboard.js` 데이터 흐름 분석

## 🎯 일반 모드 vs 디버깅 모드 비교

| 구분 | 일반 Agent 모드 | 디버깅 모드 |
|------|----------------|------------|
| **목적** | 코드 작성, 기능 추가, 수정 | 오류 분석, 문제 해결 |
| **접근 방식** | 요구사항 → 구현 | 에러 → 원인 파악 → 수정 |
| **분석 깊이** | 기능 중심 | 실행 흐름 중심 |
| **출력** | 코드 변경, 새 파일 | 문제 분석 보고서, 수정 |
| **사용 시점** | 개발 초기, 기능 확장 | 오류 발생, 디버깅 필요 |

## 📝 디버깅 모드 활용 팁

### 1. 에러 로그 제공
```
브라우저 콘솔 (F12 → Console)에서 에러 메시지 복사해서 제공하면
AI가 더 정확한 분석 가능
```

### 2. 재현 단계 설명
```
1. 회원가입 → 일반회원 선택
2. 프로필 수정 → 키/몸무게 입력
3. 저장 → 대시보드로 이동
4. 문제: 신체 정보 표시 안 됨
```

### 3. 스크린샷 제공
- 에러 메시지 화면
- 예상과 다른 UI 화면
- Supabase 테이블 데이터

---

## 🔧 현재 프로젝트 상태

### 주요 수정 완료 ✅
1. 역할 번역 오류 수정 (`athlete` → '일반회원')
2. 대시보드 신체 정보 표시 개선
3. 역할 배지 색상 구분
4. 번역 누락 부분 추가
5. 성능 최적화 (병렬 데이터 로딩)

### 디버깅이 필요한 경우 📋
1. 회원가입 후 데이터가 Supabase에 저장 안 되는 경우
2. 로그인 후 무한 로딩 발생
3. 출석 체크 실패
4. 역할이 잘못 표시됨
5. API 호출 타임아웃

---

## 🎓 결론

**디버깅 모드는:**
- ✅ 에러가 발생했을 때 원인을 찾는 데 매우 유용
- ✅ 복잡한 버그의 근본 원인을 파악할 때 강력
- ❌ 일반적인 기능 개발이나 코드 작성에는 불필요

**현재 상황 (SQL 에러):**
- 일반 Agent 모드에서도 충분히 해결 가능
- 마크다운 파일 전체를 붙여넣은 실수 → 순수 SQL 파일 제공으로 해결
- 굳이 디버깅 모드로 전환할 필요 없음

**추천:**
현재는 일반 Agent 모드를 유지하고, 만약 앞으로 복잡한 런타임 에러나 데이터 흐름 문제가 발생하면 그때 디버깅 모드를 활용하세요!
