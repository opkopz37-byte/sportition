# 스킬 체인 기획: 레벨 체인지 배잽 (잽 → 배잽 분기)

> **아웃복싱형** 상단은 **`위(투)`**, **인파이팅형** 상단은 **`위(뒷손)`** 으로 구분. 하단은 공통으로 **`아래(배 앞손)`** 을 기준으로 서술합니다.

---

## 1. 공통 상위 구조

**기본**

* 잽

**파생**

* 배잽

**완성 스킬 분기**

* 아웃복싱형
* 인파이팅형

---

## 2. 스킬별 정리

### 스킬 A. 레벨 체인지 배잽 - 아웃복싱형

* **트리**: 잽 → 배잽 → 레벨 체인지 배잽
* **동작**: 아래(배 앞손) → 위(투)
* **설명**: 배를 먼저 건드려 가드를 아래로 유도한 뒤, 상단 투로 연결하는 높이 변화형 연계. 거리 유지 상태에서 견제 후 상단 연결에 적합한 아웃복싱용 스킬.

**양식형 표기**

```text
[이름]
레벨 체인지 배잽 - 아웃복싱형

[트리]
잽 → 배잽 → 레벨 체인지 배잽

[동작]
아래(배 앞손) → 위(투)

[설명]
하단 견제로 시선을 분산시킨 뒤 상단 투로 연결하는 높이 변화 연계.
거리 유지 상황에서 사용하기 좋은 아웃복싱 스킬.
```

---

### 스킬 B. 레벨 체인지 배잽 - 인파이팅형

* **트리**: 잽 → 배잽 → 레벨 체인지 배잽
* **동작**: 아래 → 위(뒷손)
* **설명**: 하단 타격 이후 바로 상단 뒷손으로 이어지는 압박형 연계. 근거리에서 짧고 강하게 연결하는 인파이팅용 스킬.

**양식형 표기**

```text
[이름]
레벨 체인지 배잽 - 인파이팅형

[트리]
잽 → 배잽 → 레벨 체인지 배잽

[동작]
아래 → 위(뒷손)

[설명]
하단 공격으로 반응을 끌어낸 뒤 상단 뒷손으로 바로 연결하는 연계.
근거리 압박 상황에서 사용하기 좋은 인파이팅 스킬.
```

---

## 3. 더 직관적으로 한 줄 요약하면

* **잽 → 배잽 → 아래(배 앞손) → 위(투)** = 아웃복싱형
* **잽 → 배잽 → 아래 → 위(뒷손)** = 인파이팅형

---

## 4. 가장 깔끔한 최종 표기 (UI·문서)

```text
잽
└─ 배잽
   ├─ 레벨 체인지 배잽(아웃복싱형)
   │  └─ 아래(배 앞손) → 위(투)
   └─ 레벨 체인지 배잽(인파이팅형)
      └─ 아래 → 위(뒷손)
```

**구조 한 줄 (기본·파생·비트)**

* 잽(기본) → 배잽(파생) → [아래(배 앞손) · 위(투)] — 아웃복싱
* 잽(기본) → 배잽(파생) → [아래 · 위(뒷손)] — 인파이팅

---

## 5. 표기 메모 (기획·DB 반영 시)

| 항목 | 아웃복싱형 | 인파이팅형 |
|------|------------|------------|
| 첫 비트 | 아래(배 앞손) | 아래(동일 개념, 표기는 `아래`로 단순화 가능) |
| 둘째 비트 | 위(**투**) | 위(**뒷손**) |
| 거리감 | 거리 유지·견제 후 상단 | 근거리 압박·짧은 연결 |

DB `map_subtitle` 예: `아래(배 앞손) → 위(투)` / `아래 → 위(뒷손)` · 배잽 노드는 `잽(기본) → 배잽(파생)`.
