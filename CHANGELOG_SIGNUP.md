## 📦 업데이트 내용

### 1. ✅ 2단계 회원가입 플로우

기존 단일 페이지 회원가입을 **2단계 프로세스**로 개선했습니다.

#### Step 1: 계정 생성 및 인증
- ✅ 역할 선택 (일반회원/선수/체육관)
- ✅ 이메일 입력
- ✅ 비밀번호 설정 (강화된 보안 요구사항)
- ✅ 비밀번호 확인
- ✅ 약관 동의 체크박스
  - [필수] 이용약관
  - [필수] 개인정보 수집 및 이용 (핸드폰 번호 포함)

#### Step 2: 프로필 정보 입력
역할에 따라 **동적으로 입력 필드가 변경**됩니다.

**공통 필드:**
- 닉네임 ✅
- 핸드폰 번호 ✅
- 생년월일 ✅
- 성별 ✅

**일반회원/선수:**
- 키, 몸무게 (선택)
- 복싱 스타일 (선수만, 선택)
- 소속 체육관 (선택)
- 멤버십 타입 ✅

**체육관:**
- 체육관 이름 ✅
- 체육관 위치 (주소) ✅
- 대표 연락처 ✅

  - 강함 (초록)

- ✓ 비밀번호 일치 여부 실시간 표시

## 🧪 테스트 가이드

### 테스트 시나리오 1: 일반회원 가입

1. **Step 1**
   - 역할: "일반회원" 선택
   - 이메일: test@example.com
   - 비밀번호: Test1234!
   - 비밀번호 확인: Test1234!
   - 약관 동의: 모두 체크
   - "다음 단계" 클릭

2. **Step 2**
   - 닉네임: 테스트유저
   - 핸드폰: 010-1234-5678
   - 생년월일: 1990-01-01
   - 성별: 남성
   - 키: 170 (선택)
   - 몸무게: 70 (선택)
   - 소속 체육관: 스포티션 (선택)
   - 멤버십: 베이직
   - "계정 만들기" 클릭

3. **예상 결과**
   - ✅ 회원가입 성공 메시지
   - ✅ 로그인 페이지로 이동
   - ✅ DB에 데이터 저장 확인

### 테스트 시나리오 2: 선수 가입

1. **Step 1**
   - 역할: "선수" 선택
   - (나머지 동일)

2. **Step 2**
   - (공통 필드 동일)
   - 복싱 스타일: 아웃복서
   - (나머지 동일)

### 테스트 시나리오 3: 체육관 가입

1. **Step 1**
   - 역할: "체육관" 선택
   - (나머지 동일)

2. **Step 2**
   - 닉네임: 스포티션
   - 핸드폰: 02-1234-5678
   - 생년월일: 2020-01-01
   - 성별: 남성
   - 체육관 이름: 스포티션 복싱 체육관
   - 체육관 위치: 서울시 강남구 테헤란로 123
   - 대표 연락처: 02-1234-5678
   - "계정 만들기" 클릭

### 테스트 시나리오 4: 유효성 검사

**비밀번호 강도:**

```

├── 이름
├── 이메일

├── 생년월일
```

```
├── 공통 (모든 역할)
│   ├── 닉네임 ⭐
```

## 💡 구현 결정사항

### 1. 이메일 인증: 일단 생략
**결정:** Phase 1에서는 제외
**이유:**
- Supabase 기본 이메일 확인 기능 사용 가능
- 6자리 코드는 커스텀 구현 필요 (추가 개발 시간)
- MVP 출시 우선

**대안:**
- Supabase 이메일 링크 확인 사용
- 필요 시 나중에 추가

### 2. SMS 인증: 결제 시점으로 연기
**결정:** 가입 시에는 번호만 수집
**이유:**
- 개인정보보호법 준수 (약관 동의로 해결)
- SMS 발송 비용
- 토스 결제에서 본인 인증 가능

### 3. 건너뛰기 기능: 부분 적용
**결정:** 키, 몸무게, 복싱 스타일, 체육관은 선택
**이유:**
- 가입 진입 장벽 낮춤
- 필수는 최소화

### 4. 체육관 주소: 텍스트 입력
**결정:** 일단 주소 API 없이 텍스트
**이유:**
- 카카오 주소 API는 나중에 추가
- MVP에서는 간단하게
